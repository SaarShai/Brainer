# Brainer project goals — the six purposes every change must serve

User-stated (2026-07-19) canonical purpose of the Brainer skills, for any
project they are installed in:

1. Optimize token-use efficiency.
2. Maximize agent efficiency and reliability.
3. Facilitate learning and self-improvement.
4. Manage memory and the wiki.
5. Operationalize loops.
6. Provide agents with instructions, suggestions, mindsets and thinking
   guidelines.

Registered in the resident boot block of `CLAUDE.md` / `AGENTS.md` /
`GEMINI.md` (outside the install.sh sentinels, so it survives re-install).

Scope rule that follows: a change that adds machinery without clearly serving
one of these — or that costs more tokens/complexity than it saves — is out of
scope. Prefer removing over adding. The user has explicitly flagged
over-engineering/bloat as the top risk to these goals (same directive).
